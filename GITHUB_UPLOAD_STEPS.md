# GitHub Upload Steps

## Recommended repository name

```text
aws-ec2-backup-lambda-terraform
```

## Recommended repository description

```text
Serverless AWS EC2 backup automation using Terraform, Lambda, EventBridge, S3, SNS, CloudWatch, GitHub Actions, and optional GenAI backup summaries.
```

## Upload using GitHub website

1. Create a new empty GitHub repository.
2. Do not add README, `.gitignore`, or license during repo creation.
3. Extract this ZIP file.
4. Open the extracted project folder.
5. Select all files and folders inside the folder.
6. Drag and drop them into GitHub upload page.
7. Commit with message: `Initial commit - AWS EC2 backup automation`.

This project has fewer files than your Docker portfolio project, so GitHub web upload should work normally.

## Upload using Git commands

```bash
git init
git add .
git commit -m "Initial commit - AWS EC2 backup automation"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/aws-ec2-backup-lambda-terraform.git
git push -u origin main
```
